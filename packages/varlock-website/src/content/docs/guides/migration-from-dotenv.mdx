`varlock` makes it easy to migrate from `dotenv`. 

## Why Migrate?

Benefits of migrating to Varlock:
- Type safety for environment variables
- Better validation
- Improved security features

## Migration Steps

There are a few ways to migrate:

### Option 1: using `overrides`

1. Run [`varlock init`](/reference/cli#init) 
2. Update your package manager to use `varlock` instead of `dotenv` via an `override`
